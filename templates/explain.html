Matrix Factorization — The Core Math

Breaking one big matrix into two smaller, meaningful ones

Matrix Factorization says: "A big sparse matrix can be approximated as the product of two dense matrices." In our case:

R̂ = Q × Pᵀ Where: Q = User-Tag matrix (users × tags) ← learned by the GA P = Item-Tag matrix (items × tags) ← built from product features R̂ = Predicted ratings (users × items) ← what we want

The P Matrix — Tag Genome (Item Feature Vectors)

How each product is described as a numeric vector

The original paper uses MovieLens tag genome with 1128 tags like "suspenseful", "visually stunning", etc. — each tag scored 0–1 for every movie. Our system builds an equivalent for products using measurable features:

Tag Dimension How It's Computed Value Range What It Encodes
Electronics Clothing ... (one per category) 1.0 if product belongs to this category, else 0.0 0 or 1 Category membership (one-hot encoding)
Price-Level product_price / max_price_in_dataset [0, 1] Relative price — is it cheap or expensive?
View-Count total_views_for_product / max_views [0, 1] How popular (views) is this product?
Click-Rate total_clicks_for_product / max_clicks [0, 1] How often do users click on it?
Purchase-Rate total_purchases_for_product / max_purchases [0, 1] How often does viewing lead to buying?

The Chromosome — How the GA Represents Solutions

Each candidate solution is an entire Q matrix encoded as a chromosome

In a Genetic Algorithm, each candidate solution is called a chromosome. In ETagMF, one chromosome = one complete Q matrix, which represents the tag affinities for all users simultaneously.

Chromosome = Q matrix = [ User 1 row: [0.72, 0.15, 0.88, 0.33, 0.61, 0.50, 0.29, 0.74, 0.12], User 2 row: [0.08, 0.91, 0.44, 0.67, 0.23, 0.85, 0.56, 0.38, 0.77], User 3 row: [0.55, 0.42, 0.19, 0.80, 0.73, 0.11, 0.96, 0.60, 0.34], ... ] Each row = one user's learned taste profile (values ∈ [0,1]) Each column = one tag dimension Population = multiple such chromosomes (Np chromosomes total)

Fitness Function — How Good Is a Chromosome?

RMSE measures how closely predictions match actual ratings

The fitness of a chromosome (a Q matrix) is measured by how accurately it predicts the known ratings. We use RMSE — Root Mean Squared Error. Lower RMSE = better chromosome.

Step 1: For each known rating (user u, item i): predicted = dot(Q[u], P[i]) = Σ (Q[u][t] × P[i][t]) for all tags t Step 2: Normalize actual rating to [0,1] (since Q·Pᵀ ∈ [0,1]): actual_norm = (actual_rating - 1) / 4 Step 3: Compute RMSE over all known (u, i) pairs: RMSE = √( mean( (actual_norm - predicted)² ) ) Step 4: Fitness (for selection) = 1 / RMSE (lower error = higher fitness)
Why normalize ratings? The dot product Q·P lands in [0,1] because all values in Q and P are in [0,1]. But actual ratings are in [1,5]. So we normalize ratings to [0,1] using (r−1)/4 before computing RMSE, bringing both scales together.
Worked example: comparing one prediction

Repeat steps 3–6 for Ni generations. The population gradually evolves toward better Q matrices (lower RMSE). After all generations, the best Q ever seen is used to generate recommendations.
  1. Initialization — Seed the population
    Create Np chromosomes (e.g., 20). Each chromosome is a full Q matrix with all genes drawn uniformly at random from [0,1]. This gives the GA a diverse starting point to explore from.
  2. Fitness Evaluation — Score every chromosome
    For each of the Np chromosomes, compute RMSE(R, Q×Pᵀ). Store the RMSE of each chromosome. Track the globally best chromosome so far.
  3. Roulette Wheel Selection — Pick two parents
    Assign each chromosome a selection probability proportional to 1/RMSE (the lower the error, the higher the chance of being selected). Spin the roulette wheel twice to get Parent 1 and Parent 2.
  4. Row Crossover — Combine parents
    With probability Pc (default 0.8): pick a random row index (a random user's row), then swap that row between Parent 1 and Parent 2 to create two children. With probability (1-Pc): children are exact copies of parents.
  5. Mutation — Add randomness to avoid stuck solutions
    For every gene in every child: with probability Pm, replace the gene with a completely new random value in [0,1]. This prevents the population from converging too early (premature convergence).
  6. Elitism — Preserve the best solution found
    The best chromosome seen so far is always carried into the next generation unchanged. This guarantees the GA never loses its best solution.
  7. Repeat until Ni generations are complete
    Go back to step 3. Each generation the population gradually improves. The RMSE chart on the main page shows this convergence curve.

Roulette Wheel Selection — Detailed Walkthrough

How chromosomes compete to be parents based on their fitness

The roulette wheel gives each chromosome a "slice" of the wheel proportional to its fitness (1/RMSE). Spinning the wheel (picking a random number in [0, total_weight]) selects a chromosome — better ones have bigger slices.

Given chromosomes with RMSE values: Chrom A: RMSE = 0.10 → weight = 1/0.10 = 10.0 (best!) Why row crossover? A "row" in Q represents one user's full preference profile. Swapping a user row means: "combine the good preference model of one chromosome's users with the good preference model of another chromosome's users." This is biologically analogous to combining traits from two parents.

Mutation — Keeping Diversity Alive

Randomly altering genes to explore new areas of the solution space

After crossover, each gene in the child chromosome has a small probability Pm of being replaced by a brand-new random value in [0,1]. This prevents the population from getting stuck in a local optimum.

For each gene g in the chromosome: if random() < Pm: g_new = random() ← completely new random value in [0,1] else: g_new = g ← keep original gene

Mutation rate trade-off: Too high Pm (e.g., 0.5) → random search, slow convergence. Too low Pm (e.g., 0.001) → premature convergence, stuck in local optima. The paper uses 0.5 for MovieLens (1128 tags), but our system uses a smaller default (0.05) since we have far fewer tag dimensions.

Making Recommendations — The Final Step

Using the learned Q matrix to predict scores for unseen items

Once the GA converges and we have the optimal Q matrix, generating recommendations for a target user is a simple dot-product operation:

  1. Extract the user's row from Q
    q_row = Q[user_index] — this is the learned preference vector for the target user.
  2. Find all products the user has NOT already rated
    We skip products already in R for this user — no point recommending what they've already reviewed.
  3. Compute a predicted score for each unrated product
    score(product_i) = dot(q_row, P[product_i]) = Σ q_row[t] × P[i][t] for all tag dimensions t.
  4. Sort by predicted score (descending) and return top-N
    The products with the highest predicted scores are recommended. These are the items whose feature vector best matches the user's learned taste profile.
Interpretation of scores: A score close to 1.0 means the user's tag affinities strongly align with the product's tag values. A score near 0 means little alignment. The absolute value matters less than the ranking — we want the top-N relative to each other.

GA Parameters — What They Control

Understanding every knob you can turn on the main page

Parameter Paper Notation What It Controls Effect of Higher Value Paper Default
Population Size Np Number of chromosomes (Q matrices) in each generation More diversity, better exploration — but slower per generation 20 – 400
Generations Ni (MaxGeneration) How many evolution cycles to run More refinement, lower RMSE — but longer runtime 200 – 4000
Crossover Prob. Pc Probability that two parents exchange a row More gene mixing, faster convergence but more disruption 0.8
Mutation Prob. Pm Probability each gene is randomly replaced More exploration, avoids local optima — too high = random walk 0.5
Random Seed — Starting state for all random number generation Different seed = different (but comparably good) result 42
